Архив содержит перечень задач, подлежащих выполнению в рамках образовательного процесса.

Перечень задач:

1. Создание репозитория в системе контроля версий GitHub и добавление пользователя art2857.
2. Инициализация репозитория с директорией src/tasks, предназначенной для хранения исходных данных задач. По мере выполнения задач данные должны переноситься в директорию src/answer. (Первый PR)
3. Настройка конфигурационных файлов: tsconfig, eslint и prettier. (Второй PR)
4. Использование методологии разработки Gitflow (https://skyeng.ru/magazine/wiki/it-industriya/git-flow-chto-takoe/) для каждой задачи. Это предполагает создание отдельной ветки для каждой задачи и формирование pull request (PR). Ссылка на PR должна быть отправлена ответственному лицу. После выполнения каждой задачи должен быть проведен код-ревью, в ходе которого будут выявлены и обозначены недочеты, требующие исправления. Запрещается самостоятельное слияние PR в основную ветку (main). 

Успешное завершение задач: все задачи должны быть выполнены, успешно пройдены процедуру код-ревью и интегрированы в основную ветку (main).
